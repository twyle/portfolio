About

I am a self-starter, once I'm clear on what the task or project is, I am good at determining the best way to accomplish it. I'm also pro-active and actively evaluate current procedures and processes to identify any foreseable issues or concerns. I find it easier to move to plan B when a solid plan A is in place. I am a team player and actively look for opportunities with collaborate with other people especially A players.

I previously worked at an agritech startup that built an android application that enabled farmers to scan their crops for diseases then offer suggestions on what to do incase of an infection such as where to find an agrovet for professional consultation or pesticides to purchase. I was the backend developer and cofounder. I designed the backend architecture, a mixture of two microservices decoupled using redis queues. I also collaborated with our data scientists to build a data processing pipeline that used redis workers to preprocess images and analyze them for diseases as well as a search engine that used elasticsearch to answer farmers questions. I also built a custom library for uploading images to Amazon Web Service S3 and set up a storage scheme that kept track of image metadata such as timestamp and location for easier further analysis. I also did technical documentation, mainly for internal use by new hires but also enabled easier intergration by third party APIs.

I studied Mechatronics Engineering, focusing more on automation.I did component design using Solid Works and modelling using Matlab.

The most interesting and challenging project that I have worked on is a Python library that wraps aroung the YouTube V3 API. I wanted to analyze video stats for Kenyan YouTubers, especially the comments to find out who could influence a given topic better. This was inspired by what Advanced Symbolics did using twitter data. I had to learn how the YouTube API is structured, which was a bit challenging especially the Authentication part. The library enables the user to get various statistics on YouTube resources such as Videos, Channels, Playlists and Comments. This includes dates published, views, likes, and comments. Future versions will allow upload of videos, creation of playlists, and leaving comments on videos as well as async operations. During the project I had to learn how to work with generics and design patterns to reduce code duplication.

I love working in fast-paced challenging environments especially where data sourcing and processing is involved. My other tech stack includes Amazon Web Services, Linux operating system, Postgres Database and web scraping. I especially enjoy analyzing textual data. My Recent Work

Here Are Some Of My Most Recent Solo Projects

Influencer API

This is an API that provides data on the Videos, Channels and Playlists uploaded and managed by Kenyan YouTubers. The API provides the data needed to study the creator ecosystem for the Kenyan ecosystem. This includes data points such as when resource was created, number of subscribers, likes, views and comments. It's built with Python and Flask and deployed to AWS EC2 instance in a docker container. It uses gunicorn with traefik. Its documentation is hosted on readthedocs.io. GitHub project was used as the project management tool. Github actions are used to run linters and autodeploy new releases. It is an open source project and provides contribution guideline as well as issue generation templates.

Github Live Demo API Documentation

YouTube V3 Python Wrapper (ayv)

This is a python library that wraps around the YouTube V3 API allowing the user to manage YouTube resources such as Playlists, Channels, Videos and Comments. This includes and is not limited to obtaining resource information such date published, number of views, likes, subscribers as well as comments. The library is hosted on PyPI, and its documentation is available on readthedocs.io. Github actions are used to run the linters and autodeploy new releases to PyPI. The project is hosted on Github and being open source provides a contribution guideline, with issues being tracked using Github project boards.

Github Live Demo API Documentation

Influencer API

This is a data pipeline that is used to add data to the Influncer API. The API is built with Python and FastAPI and uses Celery workers with Redis to load data from YouTube in the background. To use, provide a video id or channel id or playlist id and the api will run a background task that will fecth the details from YouTube and add them to the Influencer API.
